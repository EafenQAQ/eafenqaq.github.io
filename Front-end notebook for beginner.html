前端入门笔记



名词解释


什么是HTML?

HTML是超文本标记语言（HyperText Markup Language）的缩写，它是一种用于创建网页结构和内容的标记语言。HTML使用一系列标记（也称为标签）来描述网页的各个部分，如标题、段落、链接、图像等，并定义了这些元素在网页中的显示方式和排布方式。

HTML通过使用起始标签和结束标签来表示不同的元素，起始标签包含元素的名称，而结束标签在名称前加上斜杠（/）。例如，<p> 表示段落的起始标签，</p> 表示段落的结束标签。在起始标签中，还可以包含属性，用来提供关于元素的额外信息，如样式、链接地址等。

HTML是构建万维网的基础，它是网页内容的骨架，定义了文本、图片、链接等如何在浏览器中呈现和交互。除了结构化文档内容，HTML还能够通过与其他技术（如CSS和JavaScript）的结合，实现更丰富、动态的网页效果和交互功能。

块元素

在HTML和CSS中，块元素（Block Element）是一种布局行为特定的元素，它通常会在网页布局中自动开始新的一行，并尽可能延伸到其容器的最大宽度。块元素不仅仅是对内容的组织，还影响到页面的视觉和结构布局。

特点：

  1. 自动换行：块元素会在前后自动添加换行，因此通常不与其他元素在同一行显示。
  2. 宽度自适应：默认情况下，块元素的宽度会扩展到其父元素的宽度，除非指定了宽度。
  3. 可设置宽高：你可以通过CSS为块元素指定宽度和高度。

常见的块元素有：

  • <div>：用于页面或应用的布局和结构，是最常见的块级元素。
  • <p>：表示文本的段落。
  • <h1>，<h2>，<h3>，<h4>，<h5>，<h6>：表示不同级别的标题。
  • <ol>，<ul>：有序列表和无序列表。
  • <li>：列表项。
  • <table>：表格。
  • <header>，<footer>，<section>，<article>：用于定义文档的不同部分。

了解块元素对于掌握网页布局是非常重要的，因为它们是构建网页结构的基础。

行内元素

行内元素（Inline Element）是HTML中的一种元素类型，它与块元素的布局行为不同。行内元素不会引起内容的换行，而是可以在其他行内元素或文本内容中连续显示。行内元素通常用于对文本进行格式化或插入内容，而不破坏文档的流（flow）。

特点：

  1. 不会自动换行：行内元素通常不会在其前后创建新行，而是与周围的文本或其他行内元素连续显示。
  2. 宽度随内容变化：行内元素的宽度通常只取决于其内容。
  3. 不可设置宽高：通常，对行内元素设置宽度和高度是无效的（除非改变其显示类型，比如通过CSS的display属性）。
  4. 可包含文本和其他行内元素：行内元素可以包含文本和其他行内元素，但通常不能包含块元素。

常见的行内元素有：

  • <span>：用于对文本的小块进行组织或应用样式，没有具体语义。
  • <a>：定义超链接，用于从一个页面链接到另一个页面。
  • <img>：嵌入图像。
  • <b>，<strong>：用于文本加粗，其中<strong>还带有强调的语义。
  • <i>，<em>：用于文本斜体，其中<em>表示强调。
  • <br>：插入一个换行符。
  • <input>，<textarea>，<button>：用于表单的输入框、文本区域和按钮。

理解行内元素的特性和用途有助于更好地控制网页中的文本布局和样式，使其符合设计要求。

行内块元素

行内块元素(inline-block element)是指在文档流中既具有行内元素的特性，又具有块级元素的特性的元素。这种元素可以在一行内显示，并且可以设置宽度和高度，同时也可以接受一些块级元素的样式属性，例如边距（margin）、填充（padding）等。

一些常见的行内块元素包括：

  • <img>：图像元素
  • <input>：输入元素
  • <button>：按钮元素
  • <select>：选择框元素
  • <span>：用于设置文本样式的行内元素
  • <div>：通常是块级元素，但可以通过CSS样式设置为行内块元素

这些元素都可以在一行内显示，并且可以设置宽度和高度，以及其他块级元素的样式属性。这使得它们在布局和设计网页时非常有用。

什么是CSS?

CSS代表层叠样式表（Cascading Style Sheets），它是一种用于描述HTML或XML文档外观和样式的样式表语言。CSS用于控制网页的布局、字体、颜色、大小、间距等外观方面的样式，使得网页能够更具视觉吸引力和易读性。

与HTML相比，CSS更关注于网页的外观和样式，而不是内容结构。通过CSS，您可以轻松地控制网页的布局和外观，而无需更改HTML代码。这使得网页设计变得更加灵活，同时也更容易维护和更新。

CSS使用一种简单的选择器语法，可以选择HTML文档中的特定元素，并为其应用样式。样式规则由两部分组成：选择器和声明块。选择器指定要应用样式的HTML元素，而声明块包含一组属性-值对，定义了要应用的样式属性及其值。

例如，以下是一个简单的CSS样式规则，将段落文本颜色设置为红色：

css p { color: red; }

这个规则中，p 是选择器，表示所有段落元素，color: red; 是声明块，表示将文本颜色设置为红色。通过将这个样式规则放置在HTML文档的<style>标签内或者外部的CSS文件中，就可以应用这个样式到文档中的所有段落元素上。

“cascade”代表什么？

在层叠样式表（CSS）中，"cascade" 意味着样式规则可以根据一定的规则进行层叠和覆盖。这种层叠机制允许多个样式来源（如外部样式表、内部样式表、内联样式和浏览器默认样式）中的样式规则相互影响，并按照一定的优先级顺序来确定最终的样式效果。

具体来说，"cascade" 涉及到以下几个方面：

  1. 来源的优先级：不同来源的样式规则有不同的优先级，一般来说，内联样式（即直接在HTML标签中使用style属性指定的样式）优先级最高，其次是内部样式表（即在HTML文档的<style>标签内部定义的样式），最低优先级是外部样式表（即外部的CSS文件）。

  2. 特殊性：样式规则的特殊性（Specificity）也影响层叠规则。特殊性是一种计算规则，用于确定哪个样式规则更具体，从而具有更高的优先级。

  3. 顺序：当多个具有相同优先级和特殊性的样式规则应用于同一个元素时，后面的规则将覆盖前面的规则。这意味着在CSS文件中，后面的样式规则会覆盖前面的样式规则。

通过这种层叠机制，CSS可以实现样式的管理和调整，同时也保持了灵活性和可维护性。

CSS的三种导入方式

CSS有三种主要的导入方式：

  1. 外部样式表：外部样式表是将CSS代码保存在独立的.css文件中，并通过在HTML文档的<link>标签中引用该文件来导入样式。这种方式将样式代码与HTML文档分离，使得样式可以在多个页面之间共享，提高了代码的可维护性和重用性。

    示例：

    html <head> <link rel="stylesheet" type="text/css" href="styles.css"> </head>

  2. 内部样式表：内部样式表是将CSS代码直接嵌入到HTML文档的<style>标签内部。这种方式适用于单个文档或页面的样式定义，样式规则仅对当前文档有效。

    示例：

    html <head> <style> body { background-color: lightblue; } h1 { color: navy; } </style> </head>

  3. 内联样式：内联样式是直接在HTML标签的style属性中指定CSS样式规则。这种方式适用于单个元素的特定样式，具有最高的优先级，但不易于维护和修改。

    示例：

    html <h1 style="color: red;">Hello, World!</h1>

这三种导入方式各有优劣，开发者根据具体需求和项目特点选择适合的方式。

常见的CSS选择器

CSS选择器用于选择要应用样式的HTML元素。以下是常见的CSS选择器：

  1. 元素选择器（Element Selector）：通过元素名称选择元素。 示例：p 选中所有 <p> 元素。

  2. 类选择器（Class Selector）：通过类名选择元素。 示例：.class 选中所有具有 class="class" 属性的元素。

  3. ID选择器（ID Selector）：通过ID选择元素。 示例：#id 选中具有 id="id" 属性的元素。

  4. 通用选择器（Universal Selector）：选择文档中的所有元素。 示例：* 选中所有元素。

  5. 属性选择器（Attribute Selector）：通过属性选择元素。 示例：[attribute] 选中具有 attribute 属性的元素。

  6. 属性值选择器（Attribute Value Selector）：通过属性值选择元素。 示例：[attribute=value] 选中具有 attribute 属性且值等于 value 的元素。

  7. 后代选择器（Descendant Selector）：选择元素的后代元素。 示例：ancestor descendant 选中所有 ancestor 元素的后代 descendant 元素。

  8. 子元素选择器（Child Selector）：选择元素的直接子元素。 示例：parent > child 选中 parent 元素的直接子元素 child。

  9. 相邻兄弟选择器（Adjacent Sibling Selector）：选择元素的下一个兄弟元素。 示例：prev + next 选中 prev 元素的下一个兄弟元素 next。

  10. 兄弟选择器（General Sibling Selector）：选择元素的所有兄弟元素。 示例：prev ~ siblings 选中 prev 元素之后的所有兄弟元素 siblings。

  11. 伪类选择器（Pseudo-class Selector）：根据元素的状态或位置选择元素。 示例：:hover 选中鼠标悬停在元素上的状态。

  12. 伪元素选择器（Pseudo-element Selector）：选择元素的特定部分。 示例：::before 在元素内容之前插入内容。

    这些选择器可以单独使用，也可以组合使用，以满足不同的样式需求。

  9. none：无边框。

这些值可以单独使用，也可以组合使用，例如 border-style: dashed solid double; 将会创建一个由虚线、实线和双线组成的边框。

文档流

文档流（Document Flow）是指在HTML文档中，元素按照其在HTML中出现的顺序依次排列的方式。HTML文档中的元素按照文档流的规则依次排列，如果不进行特殊的定位或浮动处理，元素将按照从上到下、从左到右的顺序依次排列。

文档流的特点包括：

  1. 垂直排列：元素按照在HTML中出现的顺序从上到下依次排列。
  2. 水平排列：在垂直排列的基础上，元素按照在HTML中出现的顺序从左到右依次排列。
  3. 元素堆叠：在文档流中，元素会按照其在HTML中的顺序依次堆叠，后面的元素会覆盖前面的元素。
  4. 块级元素和行内元素：文档流中的元素可以分为块级元素和行内元素，块级元素会独占一行并从新行开始显示，而行内元素则按照其内容的大小进行排列，并与相邻的行内元素共享一行。
  5. 默认定位：文档流中的元素默认是按照文档流的规则进行排列，不进行特殊的定位处理时，元素会按照文档流的规则排列。

文档流在网页布局中起到基础性的作用，通过对文档流中元素的样式、布局进行调整和控制，可以实现各种不同的网页布局效果。

常用的网页布局方式

网页的布局方式主要有以下几种：

  1. 流式布局（Flow Layout）：元素按照文档流从上到下依次排列，宽度默认占据父容器的100%。这是默认的布局方式，适合简单的页面结构。

  2. 浮动布局（Float Layout）：通过float属性将元素脱离文档流，使得其他内容可以环绕在其周围。常用于创建多列布局，但需要注意清除浮动以避免布局混乱。

  3. 弹性布局（Flexbox Layout）：使用display: flex或display: inline-flex将容器变为弹性容器，子元素成为弹性项目，可以通过设置弹性项目的属性来实现灵活的布局，如水平居中、垂直居中、等分空间等。

  4. 网格布局（Grid Layout）：通过display: grid将容器变为网格容器，可以将内容划分为行和列，通过网格线来定位和对齐元素，适合复杂的布局结构。

  5. 响应式布局（Responsive Layout）：使用媒体查询（Media Queries）和相应的CSS技术，使得网页可以根据设备的不同尺寸和屏幕宽度进行自适应布局，以提供更好的用户体验。

  6. 固定布局（Fixed Layout）：通过固定宽度或高度来布局页面，不随浏览器窗口的大小变化而改变布局，适合于需要固定展示内容的页面。

  7. 混合布局（Mixed Layout）：结合以上布局方式的特点，灵活运用不同的布局方式来实现复杂的页面布局效果，如混合使用弹性布局和网格布局来创建多列响应式布局等。

选择合适的布局方式取决于页面的设计需求、内容结构和用户体验目标。

浮动布局

浮动布局（Float Layout）：通过float属性将元素脱离文档流，使得其他内容可以环绕在其周围。常用于创建多列布局，但需要注意清除浮动以避免布局混乱。

浮动的三大特性： - 脱标：脱离标准流 - 一行显示，顶部对齐 - 具备行内块元素特性

清除浮动的方法：

清除浮动（Clear Float）是为了解决浮动元素造成的父容器高度塌陷（父容器无法包裹浮动元素而导致高度为0）或者浮动元素与其他元素重叠的问题。以下是几种常用的清除浮动的方法：

  1. 使用空元素清除浮动：在浮动元素的后面添加一个空的<div>元素，并设置其样式为clear: both;。这样可以使得该空元素在浮动元素下面占据一定高度，从而清除浮动。

    html <div style="clear: both;"></div>

  2. 使用clearfix hack：通过给父容器添加一个clearfix类，并设置该类的样式来清除浮动。这是一种常见的清除浮动的方法，例如使用以下CSS代码：

    `css .clearfix::after { content: ""; display: table; clear: both; }

    .clearfix { zoom: 1; / 兼容IE浏览器 */ } `

    然后在需要清除浮动的父容器上添加clearfix类：

    html <div class="clearfix"> <!-- 浮动元素 --> <div style="float: left;"></div> <div style="float: right;"></div> </div>

  3. 使用overflow属性：给父容器添加overflow: hidden;或者overflow: auto;样式也可以清除浮动。这种方法会创建一个新的块级格式化上下文（Block Formatting Context），从而使得父容器能够包裹浮动元素。

    css .parent-container { overflow: hidden; /* 或者 overflow: auto; */ }

这些方法可以根据具体的情况选择使用，以达到清除浮动的效果。

定位布局的几个常用种类

CSS中的定位布局指的是使用定位属性来控制元素在页面中的位置。主要的定位属性有以下几种：

  1. 静态定位（static）：元素的默认定位方式，按照文档流排列，无法通过top、right、bottom、left属性来调整位置。

  2. 相对定位（relative）：相对于元素在文档流中的原始位置进行定位，可以通过top、right、bottom、left属性来调整位置，但不会影响其他元素的布局。

    css .box { position: relative; top: 20px; left: 30px; }

  3. 绝对定位（absolute）：相对于最近的具有定位属性（非static）的父元素进行定位，如果没有符合条件的父元素，则相对于浏览器窗口进行定位。通过top、right、bottom、left属性来调整位置，会脱离文档流，不会影响其他元素的布局。

    css .box { position: absolute; top: 0; right: 0; }

  4. 固定定位（fixed）：相对于浏览器窗口进行定位，始终保持在页面的固定位置，通过top、right、bottom、left属性来调整位置。

    css .box { position: fixed; bottom: 0; left: 0; }

  5. 粘性定位（sticky）：结合了相对定位和固定定位的特点，当元素在屏幕内时，表现为相对定位；当滚动超出屏幕时，表现为固定定位。

    css .box { position: sticky; top: 10px; }

这些定位属性可以灵活组合使用，配合其他CSS属性实现各种复杂的布局效果。

JavaScript的控制语句

if, else if, 和 else 是 JavaScript 中用于控制程序流程的条件语句。它们用于根据条件执行不同的代码块，实现程序的逻辑分支控制。

  1. if 语句：if 语句用于执行一个代码块，如果指定的条件为真（true）。

    javascript var score = 85; if (score >= 90) { console.log('优秀'); }

  2. else if 语句：else if 语句在 if 条件不满足时进行检查，可以用来检查多个条件，只有第一个满足条件的代码块会被执行。

    javascript var score = 85; if (score >= 90) { console.log('优秀'); } else if (score >= 80) { console.log('良好'); } else if (score >= 70) { console.log('中等'); } else { console.log('不及格'); }

  3. else 语句：else 语句用于在 if 和 else if 条件都不满足时执行的代码块。

    javascript var score = 60; if (score >= 90) { console.log('优秀'); } else if (score >= 80) { console.log('良好'); } else if (score >= 70) { console.log('中等'); } else { console.log('不及格'); }

在使用这些条件语句时，需要注意以下几点：

  • 每个条件语句都必须以 {} 包裹需要执行的代码块。
  • else if 和 else 语句可以有零个或多个，但只有第一个满足条件的代码块会被执行。
  • 条件语句可以嵌套，即在一个条件语句内部包含另一个条件语句。

条件语句是编程中常用的控制流程结构，可以根据不同的条件执行不同的代码逻辑，实现程序的灵活性和逻辑判断能力。

JS的循环语句

JavaScript 中常用的循环语句有 for 循环、while 循环、do...while 循环以及 for...in 和 for...of 循环，它们可以用于重复执行一段代码，实现程序中的重复逻辑。

  1. for 循环：用于指定循环次数的情况，可以在循环头部初始化变量、设置循环条件和更新变量值。

    javascript for (var i = 0; i < 5; i++) { console.log(i); }

  2. while 循环：在指定条件为真（true）时重复执行一段代码，循环体可能一次都不执行。

    javascript var i = 0; while (i < 5) { console.log(i); i++; }

  3. do...while 循环：先执行一次循环体，然后在指定条件为真时重复执行一段代码。

    javascript var i = 0; do { console.log(i); i++; } while (i < 5);

  4. for...in 循环：用于遍历对象的属性，将对象的每个可枚举属性作为变量遍历。

    javascript var person = { name: 'Alice', age: 30 }; for (var key in person) { console.log(key + ': ' + person[key]); }

  5. for...of 循环：用于遍历可迭代对象（如数组、字符串等），循环体将被执行该对象的每个元素一次。

    javascript var colors = ['red', 'green', 'blue']; for (var color of colors) { console.log(color); }

这些循环语句在 JavaScript 中被广泛应用，可以根据不同的场景和需求选择合适的循环方式来实现程序的重复逻辑。

=和==的区别

= 和 == 是 JavaScript 中的两个不同的操作符，分别用于赋值和比较。

  1. = 赋值操作符：

    • = 是赋值操作符，用于将右侧的值赋给左侧的变量或属性。

    • 例如：var x = 10; 将值 10 赋给变量 x。

  2. == 等于操作符：

    • == 是相等操作符，用于比较两个值是否相等。在比较时，会进行类型转换。
    • 如果两个值类型不同，会尝试将它们转换为相同的类型，然后再进行比较。
    • 例如：1 == '1' 会返回 true，因为 '1' 会被转换为数字 1。

需要注意的是，相等操作符 == 进行比较时会进行类型转换，可能会导致一些意外的结果。因此，通常推荐使用严格相等操作符 === 进行比较，它不会进行类型转换，且要求比较的两个值不仅要相等，还要类型一致。

示例：

  • 1 == '1' 返回 true，因为 '1' 被转换为数字 1。
  • 1 === '1' 返回 false，因为严格相等操作符要求值不仅相等，还要类型一致。

因此，对于比较操作，推荐使用严格相等操作符 === 或 !==，以避免类型转换带来的问题。

事件

JavaScript 中常见的事件包括：

  1. 点击事件 (click event)：当用户点击某个元素时触发。
  2. 鼠标移动事件 (mousemove event)：当用户移动鼠标时触发。
  3. 键盘事件 (keyboard events)：包括按键按下 (keydown)、按键释放 (keyup)、按键持续按下 (keypress) 等。
  4. 表单事件 (form events)：包括提交表单 (submit)、重置表单 (reset)、输入框聚焦 (focus)、输入框失焦 (blur) 等。
  5. 加载事件 (load events)：包括页面加载完成 (load)、图片加载完成 (load)、音频/视频加载完成 (load) 等。
  6. 变化事件 (change events)：当元素的值或状态发生变化时触发，如输入框内容改变 (change)、复选框选中状态改变 (change) 等。
  7. 滚动事件 (scroll events)：当用户滚动页面时触发。
  8. 资源加载事件 (resource events)：包括图片加载 (load)、音频/视频加载 (load)、字体加载 (load) 等。
  9. 窗口大小改变事件 (resize event)：当浏览器窗口大小改变时触发。
  10. 离开页面事件 (unload event)：当用户离开页面时触发，可以用于执行一些清理工作。

这些事件可以通过 JavaScript 的事件监听器 (Event Listeners) 来捕获并执行相应的操作。

当使用 addEventListener 方法时，需要将事件的类型以字符串的形式传递给方法，以告诉浏览器要监听哪种类型的事件。这个事件类型字符串就是 'click'，因此需要用引号包裹起来，以表示它是一个字符串字面量而不是变量或其他表达式。

常用的事件绑定方法

事件绑定是前端开发中非常常见的操作，尤其是当需要响应用户交互时。以下是几种常用的事件绑定方式的代码示例：

1. 使用 HTML 属性绑定事件

直接在 HTML 元素中使用事件属性绑定事件处理函数。 html <button onclick="alert('Button clicked!')">Click me</button>

2. 使用 JavaScript 的 addEventListener 方法

这是现代浏览器中推荐的事件绑定方式，具有更好的灵活性和兼容性。 `javascript // 选择要绑定事件的元素 const button = document.querySelector('button');

// 定义事件处理函数 function handleClick() { alert('Button clicked!'); }

// 使用 addEventListener 方法绑定事件 button.addEventListener('click', handleClick); `

3. 直接在 JavaScript 中使用事件属性

这种方式较为简单，但不如 addEventListener 灵活。 `javascript // 选择要绑定事件的元素 const button = document.querySelector('button');

// 定义事件处理函数 button.onclick = function() { alert('Button clicked!'); }; `

4. jQuery 事件绑定

如果使用 jQuery 库，可以通过 on 方法绑定事件。 javascript // 使用 jQuery 选择元素并绑定事件 $('button').on('click', function() { alert('Button clicked!'); });

5. 事件代理（Event Delegation）

事件代理是一种优化事件处理的方法，特别适用于动态添加的元素。将事件绑定到父元素上，通过事件冒泡机制处理子元素的事件。 `javascript // 选择父元素 const container = document.querySelector('.container');

// 定义事件处理函数 function handleItemClick(event) { if (event.target.tagName === 'BUTTON') { alert('Button clicked!'); } }

// 绑定事件到父元素 container.addEventListener('click', handleItemClick); `

6. 一次性事件绑定

只希望事件处理函数执行一次，可以使用 once 选项。 `javascript // 选择要绑定事件的元素 const button = document.querySelector('button');

// 定义事件处理函数 function handleClick() { alert('Button clicked!'); }

// 使用 addEventListener 方法绑定一次性事件 button.addEventListener('click', handleClick, { once: true }); `

这些示例涵盖了不同场景下的事件绑定方式，可以根据实际需求选择合适的方法来实现事件处理。

API 是应用程序编程接口（Application Programming Interface）的缩写，其英文全称是 Application Programming Interface。API 是一组定义了软件组件之间如何互相通信、交互的规则和约定，它允许不同的软件系统或应用之间进行数据交换和功能调用，从而实现不同系统之间的集成和协作。API 可以用于访问操作系统、库、框架、服务等，使得开发人员能够利用已有的功能来构建新的应用程序或扩展现有的应用程序功能。

响应式布局

响应式布局（Responsive Layout）是一种网页设计方法，旨在通过使用流动网格和灵活的图像，以适应不同设备和屏幕尺寸，从而提高用户在各种设备上的浏览体验。具体来说，响应式布局使得网站可以在台式机、平板电脑和智能手机等设备上都能良好显示，无需为每种设备分别设计和开发独立的版本。

主要特点和技术

  1. 流动网格布局（Fluid Grid Layouts）：

    • 使用相对单位（如百分比）而不是固定单位（如像素）来定义网页元素的宽度、内边距和边距。

    • 这样布局会根据屏幕大小自动调整，确保内容在不同设备上均能合理排列。

  2. 灵活的图像和媒体（Flexible Images and Media）：

    • 使用CSS来定义图像和其他媒体的最大宽度，使其根据容器的大小进行调整。

    • 例如，使用 max-width: 100%; 让图像在容器内不超过100%的宽度。

  3. 媒体查询（Media Queries）：

    • 通过CSS3的媒体查询，根据不同的屏幕尺寸和分辨率应用不同的样式。
    • 使用 @media 规则，可以针对特定条件（如设备宽度）定义样式。例如，@media (max-width: 600px) { ... } 针对宽度小于600像素的设备应用特定样式。

示例代码

以下是一个简单的响应式布局示例：

html <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Responsive Layout Example</title> <style> body { margin: 0; font-family: Arial, sans-serif; } .container { display: flex; flex-wrap: wrap; gap: 10px; padding: 10px; } .box { flex: 1 1 200px; padding: 20px; background-color: lightblue; text-align: center; } @media (max-width: 600px) { .container { flex-direction: column; } } </style> </head> <body> <div class="container"> <div class="box">Box 1</div> <div class="box">Box 2</div> <div class="box">Box 3</div> </div> </body> </html>

解释

  • 流动网格布局：在 .box 类中使用 flex: 1 1 200px; 来设定每个盒子的弹性行为，使其在不同屏幕尺寸下灵活调整。
  • 灵活的图像和媒体：如果有图像，可以通过 img { max-width: 100%; height: auto; } 确保图像不会超出其容器的宽度。
  • 媒体查询：通过 @media (max-width: 600px)，当视口宽度小于等于600像素时，将 .container 的方向调整为列方向。

总结

响应式布局通过使用流动网格、灵活的图像和媒体查询，确保网页在各种设备上均能良好显示，提高用户体验，减少开发和维护多版本网站的工作量。

=的功能

在编程语言中，= 通常用作赋值运算符，但它在不同的上下文中可以有不同的作用。以下是 = 运算符在 JavaScript 中的主要作用：

  1. 赋值运算:

    • 基本用途: 将右侧的值赋给左侧的变量。 javascript let a = 10; // 将 10 赋值给变量 a

    • 链式赋值: 可以同时给多个变量赋值。 javascript let a, b, c; a = b = c = 5; // 将 5 赋值给 a、b 和 c

  2. 默认参数值:

    • 在函数定义中，= 用于设置参数的默认值。 javascript function greet(name = "Guest") { console.log("Hello, " + name); } greet(); // 输出: Hello, Guest

  3. 解构赋值:

    • 对象和数组的解构赋值使用 = 来将解构的值赋给变量。 `javascript // 数组解构赋值 let [x, y] = [1, 2]; // 将 1 赋值给 x，将 2 赋值给 y

      // 对象解构赋值 let {a, b} = {a: 3, b: 4}; // 将 3 赋值给 a，将 4 赋值给 b `

  4. 事件处理程序赋值:

    • 将一个函数引用赋值给事件处理程序属性（如 window.onresize）。 javascript window.onresize = resetHTMLFontsize; // 当窗口大小变化时调用 resetHTMLFontsize 函数

  5. 计算属性名赋值:

    • 在对象字面量中，计算属性名可以使用 = 进行赋值。 javascript let propName = "foo"; let obj = {[propName]: "bar"}; // 创建一个属性名为 'foo' 值为 'bar' 的对象

  6. 运算符组合:

    • 在一些高级用法中，= 可以结合其他运算符使用（如 +=、-=、*= 等），以简化赋值操作。 javascript let num = 5; num += 3; // 相当于 num = num + 3，结果是 8

总之，= 的主要作用是进行赋值，但在不同的上下文中可以用来简化代码和实现各种功能。

flex弹性布局

Flex 布局（Flexbox）是一种现代的 CSS 布局方式，用于创建灵活且响应式的布局结构。它提供了更加直观和便捷的方法来控制元素在容器中的布局方式、对齐方式和排列顺序。以下是 Flex 布局的基本概念和常用属性：

基本概念：

  1. 容器（Container）：拥有了 display: flex; 或 display: inline-flex; 属性的元素称为 Flex 容器。
  2. 项目（Item）：Flex 容器内的每个子元素称为 Flex 项目。

Flex 容器属性：

  1. display：指定元素的显示类型，可以设置为 flex 或 inline-flex。
  2. flex-direction：设置项目的排列方向，包括 row（水平方向，默认值）、row-reverse（反向水平）、column（垂直方向）和 column-reverse（反向垂直）。
  3. flex-wrap：设置项目是否换行，包括 nowrap（不换行，默认值）、wrap（换行）、wrap-reverse（反向换行）。
  4. flex-flow：flex-direction 和 flex-wrap 的缩写属性，例如 flex-flow: row wrap;。
  5. justify-content：设置项目在主轴上的对齐方式，包括 flex-start（起始位置，默认值）、flex-end（末尾位置）、center（居中对齐）、space-between（两端对齐）、space-around（每个项目周围均匀分布）和 space-evenly（项目之间和周围均匀分布）。
  6. align-items：设置项目在交叉轴上的对齐方式，包括 stretch（默认值，拉伸填满）、flex-start（起始位置对齐）、flex-end（末尾位置对齐）、center（居中对齐）和 baseline（基线对齐）。
  7. align-content：设置多行项目在交叉轴上的对齐方式，仅当项目在多行时有效，包括 flex-start、flex-end、center、space-between、space-around 和 stretch。

Flex 项目属性：

  1. order：设置项目的排列顺序，数值越小越靠前，默认值为 0。
  2. flex-grow：设置项目的放大比例，默认为 0，表示不放大，大于 0 时按比例放大。
  3. flex-shrink：设置项目的缩小比例，默认为 1，表示等比例缩小。
  4. flex-basis：设置项目的初始大小，可以设置为固定值（如 100px）或百分比（如 50%）。
  5. flex：flex-grow、flex-shrink 和 flex-basis 的缩写属性，例如 flex: 1 1 0;。
  6. align-self：设置单个项目在交叉轴上的对齐方式，覆盖容器的 align-items 属性。

示例代码：

`css .container { display: flex; / 定义为 Flex 容器 / flex-direction: row; / 设置主轴方向为水平 / justify-content: space-between; / 主轴上的对齐方式为两端对齐 / align-items: center; / 交叉轴上的对齐方式为居中对齐 / }

.item { flex: 1; / 项目会占据剩余空间 / } `

Flex 布局具有强大的灵活性和响应性，适用于各种布局场景，如导航菜单、排版布局、卡片布局等。通过灵活运用上述属性，可以轻松实现各种复杂的布局效果。
